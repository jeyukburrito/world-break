# Handoff Documents

티켓당 spec 1개 + result 1개를 작성한다.

## 네이밍
- `T-xxx-spec.md` — Claude 작성
- `T-xxx-result.md` — Codex 작성

## Spec 필수 항목
- Goal
- Scope
- Out of Scope
- Inputs
- Constraints
- Done Definition
- Expected Output

## Result 필수 항목
- Summary
- Files Changed
- Implemented
- Test Results
- Known Issues
- Risks

Codex가 result를 작성해도 완료가 아님 — review 후 승인으로 결정된다.

모든 handoff 문서는 첫 줄에 `Author: [Role/Name]`을 포함한다.
