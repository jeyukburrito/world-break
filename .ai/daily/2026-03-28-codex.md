# 2026-03-28-codex

## T-022 OAuth Callback URL Fix
- Investigated `redirectTo`, `window.location.origin`, `VERCEL_URL`, `NEXT_PUBLIC_VERCEL_URL`, `/auth/callback`, and `signInWithOAuth` usage across the repo.
- Confirmed the login callback was built in `app/login/actions.ts` from request headers and could inherit a non-canonical Vercel host.
- Added a canonical auth callback origin helper in `lib/env.ts` so production OAuth always uses `https://world-break.vercel.app` while localhost keeps its local callback URL.
- Updated `app/login/actions.ts` to build `redirectTo` from the new helper.
- Updated `docs/SUPABASE_SETUP.md` and `docs/DEPLOYMENT.md` with the exact Redirect URL / Site URL values required for Supabase and Google OAuth setup.
- Updated `.ai/handoffs/T-022-spec.md` and wrote `.ai/handoffs/T-022-result.md`.
