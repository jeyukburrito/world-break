## 신규 수정 필요 사항

### 1. [HIGH] `/matches/new` 폼 필드에 `name` 속성 및 `value` 속성 누락

- **파일**: `app/matches/new/page.tsx`
- **문제**: `myDeckId`를 제외한 모든 `<input>`/`<select>`에 `name` 속성이 없음. 서버 액션 연결 시 `formData.get()`으로 값을 읽을 수 없음. 또한 `<select>` `<option>`에 `value` 속성이 없어서 Zod enum 값과 불일치.
- **현재 코드**:

```tsx
<select className="...">
  <option>BO1</option>        <!-- formData → "BO1", Zod expects "bo1" -->
  <option>선공</option>       <!-- formData → "선공", Zod expects "first" -->
  <option>랭크전</option>     <!-- formData → "랭크전", Zod expects "ranked" -->
</select>
```

- **수정 방향**: 모든 폼 필드에 `name` 추가 + `<option>`에 enum `value` 매핑:

```tsx
<select name="matchFormat" className="...">
  <option value="bo1">BO1</option>
  <option value="bo3">BO3</option>
</select>
<select name="playOrder" className="...">
  <option value="first">선공</option>
  <option value="second">후공</option>
</select>
<select name="eventType" className="...">
  <option value="ranked">랭크전</option>
  <option value="shop">매장</option>
  <option value="friendly">친선전</option>
  <option value="tournament">대회</option>
</select>
<input name="playedAt" type="date" ... />
<input name="opponentDeckName" type="text" ... />
<input name="wins" type="number" ... />
<input name="losses" type="number" ... />
<textarea name="memo" ... />
```

### 2. [LOW] RLS 컬럼명 camelCase 정합성 확인

- **파일**: `supabase/rls.sql`
- **문제 아님 — 확인 완료**: Prisma가 `@@map("decks")` 등으로 테이블명을 snake_case로 매핑하지만, 컬럼명은 camelCase 그대로 생성됨 (`"userId"`, `"matchResultId"` 등). RLS SQL이 `"userId"` 형태로 따옴표 사용하므로 마이그레이션 SQL과 정합.
- **주의**: Supabase Dashboard SQL Editor에서 수동 작업 시 따옴표 누락하면 `userid`로 해석되어 실패함. `rls.sql` 그대로 실행할 것.
